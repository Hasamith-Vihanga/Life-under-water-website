Feedback Page

Technical Description

The "form" tag is used to collect feedback from users. It's the container for all the form fields where users enter information.

The "input" tag is used for text fields, like for entering your name, email, and phone number. Each input has a specific type (e.g., text, email, or telephone) to make sure the user enters the correct information. The required attribute is used to make sure the user doesn't leave important fields empty.

The "textarea" tag is used for longer text like comments or feedback. It allows users to type more than one line, and we set a limit of 500 characters to avoid very long feedback.

The "select" element creates a dropdown menu for users to choose the type of feedback (e.g., general inquiry, suggestion, or complaint).

Accessibility

Implemented accessibility by using "label" elements for form fields, ensuring screen reader compatibility.

Team Page

Technical Description

Each team member is displayed using a div with the class team-member

A thumbnail (img) with an alt attribute for accessibility.

A name (p.member-name) placed inside a div.thumbnail-container.

A heading (h2) for the name and a subheading (h3) for the role.

A list (ul > li) of completed tasks to summarize contributions.

Accessibility

Alt text is provided for all images to support screen readers. The page structure ensures clear navigation.

Content Page

Technical Description

Internal Links (.internal_link): Provides quick navigation within the page by linking to different sections.

Sections (section): Divides the content into structured topics: Introduction, Causes of Endangerment, Examples of Endangered Species, and Conservation Efforts.

Images (img): Each section includes relevant images with descriptive alt attributes for accessibility.

Back to Top Button ( with class .top-button): Allows users to quickly navigate to the top of the page.

Accessibility

The page uses proper semantic elements such as (header), (nav), (section), and (footer) to improve screen reader navigation.

A navigation menu with internal links allows users to jump to different sections quickly, improving accessibility for users with mobility impairments.

Challenges and Lessons Learned

One challenge was making sure the page looked good on all screen sizes. At first, some images and text were not fitting well on small screens. I fixed this by using CSS features like max-width, flexbox, and media queries to adjust the layout. Another issue was making the page easy to use for everyone, including people who use screen readers. I added proper headings and alt text for images to improve accessibility. From these challenges, I learned that testing on different devices is important and that accessibility should always be a priority in web design.

Compliance

The page follows web standards by using proper HTML structure, clear navigation, and accessible design. It meets basic accessibility guidelines with alt text for images and proper headings. The code is validated to ensure it is error-free and works well across different devices and browsers.
